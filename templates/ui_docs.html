Usage Guidelines

Design Principles

  • Hierarchy: Use size, color, and spacing to establish visual hierarchy
  • Contrast: Ensure sufficient contrast between text and background colors
  • Balance: Distribute visual weight evenly across the interface
  • Movement: Guide the user's eye through the interface with visual flow

Component Usage

  • Buttons: Use primary for main actions, secondary for supporting actions
  • Status Colors: Green for success/good, yellow for warning/medium, red for error/poor
  • Cards: Keep content concise, use consistent spacing and typography
  • Forms: Provide clear labels and helpful placeholder text

Responsive Design

  • Mobile First: Design for mobile screens first, then scale up
  • Breakpoints: Use 700px as the main breakpoint for mobile/tablet
  • Touch Targets: Ensure buttons and interactive elements are at least 44px tall
  • Content Priority: Stack content vertically on mobile, prioritize most important information
